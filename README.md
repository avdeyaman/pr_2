# 10-13 Практическое занятие 2 Разработка статического веб-интерфейса с применением принципов семантической разметки и адаптивного дизайна

## О демонтрационном проекте
Статический веб-интерфейс для магазина NFT, разработанный с применением принципов **семантической разметки** и **адаптивного дизайна**.

## Структура проекта

### Описание структуры:
- **Папка `images`** содержит все изображения, используемые в ходе создания проекта;
- **Папка `pages`** представляет собой набор страниц приложения (авторизация и аутентификация);
- **Папка `fonts`** - содержит шрифты, используемые в проекте;
- **Файл `.gitignore`** нужен для добавления файлов, которые не должны попадать в Git-репозиторий;
- **Файл `README.md`** - файл для документации проекта.

## Набор функций
- главная страница и страница входа.

## Документация для работы с проектом
### Основные ресурсы:
- **[Tailwind CSS](https://tailwindcss.com/)** - утилитарный CSS-фреймворк, который позволяет стилизовать интерфейс с помощью готовых классов прямо в HTML. В проекте используется через CDN;
- **[Google Fonts (Jost)](https://fonts.google.com/specimen/Jost)** - шрифт Jost используется для описательного текста;
- **[Redaction Font](https://github.com/TriumphLLC/Redaction)** - семейство шрифтов Redaction (Redaction 10 и Redaction 35) используется для заголовков и навигации. Шрифты находятся в папке `fonts/`;
- **[Figma](https://www.figma.com/)** - дизайн-макеты проекта созданы в Figma;
- **[MDN HTML](https://developer.mozilla.org/en-US/docs/Web/HTML)** - HTML документация от MDN;
- **[MDN CSS](https://developer.mozilla.org/en-US/docs/Web/CSS)** - CSS документация от MDN.

## Запуск проекта
Для запуска проекта необходимо в среде разработки VS Code установить расширение **Live Server**.
